# Folder structure of LLM Wiki

./llm_wiki/inbox
Where new sources land after sync. Organized by connector:
  inbox/substack/
  inbox/youtube/
  inbox/apple-notes/
  inbox/manual/

Each source is a folder containing index.md (main content with YAML frontmatter)
plus optional attachments (images/, papers/, original.html, original.json).

Sources are moved to ./raw/ after LLM compilation via:
  cogforge inbox mark-processed <source_id>


./llm_wiki/raw
Sources after compilation, stored unchanged for provenance. Mirror of inbox layout:
  raw/substack/
  raw/youtube/
  raw/apple-notes/
  raw/manual/

Movement handled by cogforge inbox mark-processed — do not move files manually.


./llm_wiki/history
Tracks all persisted changes and sessions. Managed by the Cogforge CLI:

  history/<YYYY-MM-DD>.log    Daily changelog, one entry per line.
                              Created by cogforge wiki log --message "...".

  history/sessions/            One YAML file per substantial interaction.
                              Created by cogforge wiki session-new --summary "..." --domain "...".

Do not append to history files manually — the CLI handles timestamping and formatting.


./llm_wiki/.llmkb
Internal Cogforge state directory. Never edit directly.

  .llmkb/state/sources/        One YAML file per source tracking lifecycle (inbox →
                               processed → failed → excluded).

  .llmkb/reports/              Canonical run reports in YAML.


./llm_wiki/pageindex
Long-document index artifacts. Generated by cogforge pageindex run.

  pageindex/<connector>/<source_id>/


./llm_wiki/wiki
Compiled knowledge — what we build and maintain.

Each subfolder has a <folder>_index.md with one line per file:
  <filename> | <title> | <short description (max 100 words)>

  1. concepts/            The main knowledge type. Aggregates what we know about a
                          concept or entity. Template: templates/concepts.md

  2. domain-context/      One file per domain that future agents read first.
                          Template: templates/domain-context.md

  3. decisions/           Durable user decisions. Prevents re-litigating.
                          Template: templates/decisions.md

  4. synthesis/           Combines multiple concepts within one domain.
                          Template: templates/synthesis.md

  5. derived-outputs/     Shareable or multi-domain outputs.


./llm_wiki/system
How the wiki works. NEVER modify. Maintainer-only.

./llm_wiki/templates
Templates for new wiki pages. Agents reference specific template files.


./llm_wiki/sources.yaml
Source connector configuration. Validated by cogforge config validate.
